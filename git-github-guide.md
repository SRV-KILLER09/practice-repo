# Git & GitHub Quick Guide

## What is Git?

Git is a distributed version control system that helps developers track changes in source code and collaborate effectively.

## Common Git Commands

### Clone a Repository

```bash
git clone <repository-url>
```

### Check Repository Status

```bash
git status
```

### Create a New Branch

```bash
git checkout -b feature-branch
```

### Stage Changes

```bash
git add .
```

### Commit Changes

```bash
git commit -m "Describe your changes"
```

### Push Changes

```bash
git push origin feature-branch
```

### Pull Latest Changes

```bash
git pull origin main
```

## Pull Request Workflow

1. Fork the repository.
2. Clone your fork.
3. Create a new branch.
4. Make your changes.
5. Commit and push the changes.
6. Open a Pull Request.
7. Address review comments if required.

## Best Practices

- Write meaningful commit messages.
- Keep pull requests focused and small.
- Read the project's contribution guidelines.
- Test changes before pushing.
- Respect feedback from maintainers.

## Open Source Goals

- Learn Git and GitHub.
- Make quality contributions.
- Help other developers.
- Participate in community projects.
